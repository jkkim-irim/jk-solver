# Solver Iteration 수렴성 검증 결과

## 목적

solver iterations 및 ls_iterations 수가 접촉 안정성에 미치는 영향 확인.

## 테스트 조건

- 구(R=0.3, density=10) → 10N 힘으로 박스(half=0.5)에 충돌
- ke=10000, kd=100, cone=elliptic, impratio=1.0, dt=0.002
- 측정: 구-박스 최대 침투, 최대 법선력, 구-바닥 법선력, z좌표 진동, 잔여 속도

## 분석

### iterations

- **2회면 이미 수렴** — iter=2와 iter=200의 결과가 동일
- iter=1만 약간 다름 (침투 9.30mm vs 9.51mm, 법선력 245 vs 235.5)
- 정적 법선력(ground NF)은 iter=1에서도 정확히 mg=11.09N
- **권장: iterations >= 5** (안전 마진 포함)

### ls_iterations

- **5회면 수렴** — ls=5와 ls=30의 결과가 동일
- ls=1에서 약간 높은 법선력 스파이크 (260 vs 235N)
- **권장: ls_iterations >= 5**

### 핵심 발견

- 이 시나리오(단순 구-박스 충돌)에서는 solver가 매우 빨리 수렴
- 다관절 로봇 등 복잡한 시스템에서는 더 많은 iterations 필요할 수 있음
- 학습 환경 기준값(iter=70, ls=100)은 충분히 여유 있음
